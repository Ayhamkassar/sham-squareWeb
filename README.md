# شام بريستو — لوحة تحكم React Native

تحويل كامل للوحة تحكم المتجر من تطبيق ويب (React + Vite) إلى تطبيق **React Native** باستخدام **Expo**، مع الالتزام بمبادئ **الكود النظيف (Clean Code)**.

## طريقة التشغيل

```bash
npm install
npx expo start
```

ثم امسح رمز QR عبر تطبيق Expo Go، أو شغّل على محاكي:

```bash
npm run android   # أو
npm run ios
```

## مبادئ الكود النظيف المتبعة

- **فصل الاهتمامات (Separation of Concerns)**: كل طبقة في مجلد مستقل ومسؤولة عن شيء واحد فقط:
  - `src/types` — تعريفات TypeScript البحتة (مطابقة للنسخة الأصلية).
  - `src/theme` — الألوان والمسافات والخطوط (بديل Tailwind، بدون أي قيمة hex متناثرة في الشاشات).
  - `src/i18n` — الترجمة العربية/الإنجليزية و RTL، عبر `useLocale().t()`.
  - `src/data` — البيانات الوهمية الأولية فقط (لا منطق فيها).
  - `src/context` — **طبقة منطق الأعمال**: `storeReducer.ts` (reducer نقي بدون آثار جانبية) + `StoreContext.tsx` (يربط الـ reducer بإشعارات Toast) + `AuthContext` + `ToastContext`. هذا يفصل "ماذا يحدث في البيانات" عن "كيف يظهر ذلك للمستخدم".
  - `src/components/common` — مكوّنات UI قابلة لإعادة الاستخدام (Card، Badge، FormField، AppModal...) تمنع تكرار الكود عبر الشاشات الـ 14.
  - `src/navigation` — Drawer (بديل الشريط الجانبي Sidebar) + Stack للتفاصيل (الطلب/العميل)، مع أنواع (Types) صارمة للمسارات.
  - `src/screens` — شاشة واحدة = مسؤولية واحدة، لا تحتوي على منطق أعمال، فقط تستهلك الـ Context وتعرض المكوّنات المشتركة.

- **DRY**: منطق مثل حالة الطلب (`orderStatus.ts`)، أو الألوان الدلالية (`Badge` tones)، معرّف مرة واحدة ومُستخدم في كل الشاشات.
- **دوال نقية قابلة للاختبار**: مثل `validateCredentials` في شاشة الدخول، و`storeReducer` بالكامل — كلاهما بلا آثار جانبية ويمكن اختبارهما بمعزل عن واجهة المستخدم.
- **أسماء واضحة ومعبّرة** بدل الاختصارات الغامضة، وتعليقات توضّح "لماذا" وليس "ماذا" عند الحاجة فقط.

## ملاحظات هامة حول التحويل من الويب إلى React Native

1. **التنقل**: استبدلنا `currentScreen` (state يدوي) بـ **React Navigation** (Drawer + Stack) — وهو المعيار في React Native.
2. **الأنماط (Styles)**: لا يوجد Tailwind في React Native؛ استبدلناه بـ `StyleSheet` + محرك ثيم (`useTheme`) موحّد.
3. **RTL**: بما أن React Native لا يحتوي DOM، استُبدل `MutationObserver` الخاص بالويب بـ:
   - دالة `t()` صريحة للترجمة.
   - `I18nManager.forceRTL()` لقلب اتجاه الواجهة (يتطلب إعادة تشغيل التطبيق على الأجهزة الفعلية، وهذا سلوك طبيعي في React Native).
4. **التخزين المحلي**: البيانات تعيش حالياً في الذاكرة عبر `useReducer` (تماماً كسلوك الديمو الأصلي بدون باك-إند). لإضافة حفظ دائم لاحقاً، أضِف طبقة `AsyncStorage` أو API حقيقي داخل `StoreContext` دون لمس الشاشات.

## الشاشات المُحوَّلة (14 شاشة)

لوحة التحكم، المنتجات، الفئات، المخزون، الطلبات (+ تفاصيل الطلب)، العملاء (+ ملف العميل)، المدفوعات، الكوبونات، الإشعارات، الدعم الفني، التقييمات، الأدوار والصلاحيات، سجل النشاطات، الإعدادات — بالإضافة إلى شاشة تسجيل الدخول.
